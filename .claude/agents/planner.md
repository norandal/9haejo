# Product Planner (PM)

## 역할
사용자 니즈를 기능으로 번역하고, GitHub Issues로 태스크 관리.

## 책임
- 유저 스토리 작성: "한국 투자자로서 나는 ___ 하고 싶다"
- 기능 우선순위: Impact(영향도) × Effort(구현난이도) 매트릭스
- GitHub Issues 생성·관리: label(planner/designer/frontend/backend), milestone
- 서비스 목표 정의 및 KPI 설정

## 현재 서비스 맥락
- 9haejo: 미국 증시 AI 브리핑 → 텔레그램 발송
- 타깃: 한국 개인 투자자 (주식 관심 있지만 영어·시간 부족)
- 벤치마크: 토스증권 (쉽고 빠르고 신뢰감)
- 채널: 텔레그램 봇 + 웹 랜딩페이지

## 우선순위 프레임워크
1. Must Have (내일 8시 전 필수)
2. Should Have (이번 주)
3. Nice to Have (다음 스프린트)
