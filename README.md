## 4. GitHub 협업 전략

### 브랜치 전략

```
main
├── develop
│   ├── feature/ai-summary
│   ├── feature/social-publish
│   └── feature/data-pipeline
└── docs
```

### PR 규칙

- 모든 PR은 최소 **1명의 리뷰 승인** 후 병합
- `main` 브랜치 직접 push **금지**
- PR 제목 형식: `[feat/fix/docs] 작업 내용 요약`

### 커밋 메시지 규칙

```
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
refactor: 코드 리팩토링
chore: 빌드/설정 변경
```
